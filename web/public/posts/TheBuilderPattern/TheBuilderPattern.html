The Builder Pattern in an Application

        The builder pattern in Javascript allows us to literally build objects that only have the properties that we need, when we need them. A good example of this can be found in a project I worked on to handle recipes. After agreeing on a model that would constitute a recipe, we can then create a builder that will create one of these modeled objects for us. Here is an example of what this looks like when we are going to add a recipe into the database:

Fig. 1

You can see here that we are chaining together some function calls, which when finalized will give us a new  validated recipe object. How does this work?

If we take a look at one of the functions that make up our builder, we can get a good idea of what is allowing us to chain these functions together:

Fig. 2

Let's break down this addTitle function. As we can see, it takes in a proposed title, and then sets an object property ‘title’ to a new instance of a RecipeTitle class. This RecipeTitle class is another custom class that will sanitize our proposed title. We will look at why that is important later. For now, we see that we have a title property, and then we are returning ‘this’. What that does is it creates our chaining functionality, so that we can call multiple functions like we saw in Fig. 1  and end up with an object containing all of those properties.
