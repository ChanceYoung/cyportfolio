        Project Bakersman

        

Project Bakersman is an idea that was born out of a frustration with many modern recipe sharing websites. Our team saw a need for a place that doesn’t make you read through the author's life story before getting to the actual recipe. Not that we don’t care, but when I look for a recipe, I don’t need the backstory all the time. So that was our problem we set out to solve, and here is how we went about it.

Requirements

Our product owner brought us a set of requirements, which were very reminiscent of a recipe box full of 3x5 cards, so we took to designing our project around that idea. Let's break that down. We ultimately wanted to be able to have a user who was not authenticated to:

  • View Recipes
  • Create an Account

And a user who is authenticated will be able to:

  • View Recipes
  • Create Recipes
  • Edited created Recipes
  • Save Favorite Recipes
  • Delete their Account

Getting Started

        We got started on our api and our frontend. We chose a PERN stack for our project, as it was the stack that we were most familiar with. We forayed into the project rather blindly, slapping muddy features on one after another in an attempt to learn some of the risker features that we wanted. We fumbled a lot with trying to implement google authentication, as at the time we were learning the Oauth2 pattern in a secure coding class. It did not go as planned, and so we quickly fell into a soggy swamp of messy code, failed attempts to get it working, and other projects begging for our attention.

Coming out of the Swamp

In order to rise above this mess we had dug ourselves into, there were a few things that we did. Most particularly, I had been looking into a way to better structure my node api architecture, and applied some interesting principles that you can see by following the link below:

        A New Node Architecture

We also made sure to implement a simple continuous deployment pipeline with github actions. You can read more about that here:

Following this, our deadline approached rapidly. We had to sacrifice many hours of sleep and sanity in order to produce our final product. A lot of simplification was made, such as giving up on an Oauth pattern in favor of some primitive sessions that I implemented, which I will go over in a future post.
