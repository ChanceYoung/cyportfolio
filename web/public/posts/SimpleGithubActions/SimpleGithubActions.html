Simple Github Actions for Deployment

        In order to more easily deploy changes made to our capstone project, we implemented a github workflow.

This is the workflow file, it's quite simple, but it is extremely powerful. Let's break it down:

The “on” block allows us to state when this workflow will be triggered. It is pretty straightforward, when we push or make a successful pull request to the main branch, we are going to run the jobs that we specify.

The “jobs” block is where we can specify different jobs that we want to run on our machine. We specify a job block and it will do what we ask it to. For example, this jobs block only has one job, which we have called “build”. We specify that it runs on a particular machine, in this case a self-hosted one, which just means we have set up an action runner before. In order to that you can follow this tutorial: https://docs.github.com/en/actions/hosting-your-own-runners/adding-self-hosted-runners

Then in the “steps” we can use different actions and or commands in order to deploy what we want to deploy. The first step is to use a preexisting action that checks out our repo. And the next step runs the commands given to it.

This just goes into a folder named .github/workflows in the repo:

And when we push to our main branch, it runs the job we specified in our workflow, meaning that our site is automatically deployed with all of our recent changes.
